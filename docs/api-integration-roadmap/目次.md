# API 接続ロードマップ（目次）

## このフォルダは何？

受注登録画面（`/orders/new`）のモックデータを **Spring Boot バックエンドの REST API** に置き換えるための作業計画です。

画面エンジン側のロードマップ（`docs/screen-engine-roadmap/`）とは独立して進めます。

---

## 全体像：2つのフェーズ

現在のフロントは `src/constants/mockData.ts` にハードコードされたマスタデータを使い、保存処理は `console.info` + `alert` のモックです。これを、バックエンドの REST API 経由で H2 Database に永続化する構成に変えます。

| 順番 | ファイル | 対象 | やること | 前提 |
|------|----------|------|----------|------|
| 1 | [phase1-backend.md](./phase1-backend.md) | Spring Boot | マスタ取得 API 3本 + 受注登録 API 1本を実装する | Spring Boot プロジェクトが起動できること |
| 2 | [phase2-frontend.md](./phase2-frontend.md) | Vue（本リポジトリ） | モックデータを API 呼び出しに差し替え、保存処理を POST に変える | Phase 1 の API が動いていること |

---

## 対象 API 一覧

| # | メソッド | パス | 用途 | 認証 |
|---|---------|------|------|------|
| 1 | GET | `/api/masters/parties` | 取引先マスタ一覧（契約先・納入先の候補） | 要 |
| 2 | GET | `/api/masters/products` | 製品マスタ一覧（明細のコード選択候補） | 要 |
| 3 | POST | `/api/orders` | 受注新規登録（ヘッダ + 明細を永続化） | 要 |

※ 認証は既存の JWT（`POST /api/auth/login` → `Bearer` トークン）を使う。
